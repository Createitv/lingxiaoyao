# 模块6：进阶思维

## 本节要点

- 多轮对话策略：如何在一次对话中获得最佳结果
- 验证 Claude 的输出：避免幻觉和错误
- 了解 Claude 的局限性：知道什么不该问
- 持续学习路径：课程之后的下一步

## 多轮对话策略

和 Claude 的对话不是"一问一答"，而是**渐进式深入**：

### 三步渐进法

1. **第一轮：大方向**
   > 我想写一篇关于 [主题] 的文章，请给我一个大纲

2. **第二轮：聚焦细节**
   > 大纲很好，请展开第三部分，给出具体的例子和数据

3. **第三轮：优化打磨**
   > 请用更口语化的风格重写，并加入一个有趣的开头

### 常用追问句式

- "能给我一个具体的例子吗？"
- "请从 [角色] 的角度重新审视这个方案"
- "这个方案有什么潜在风险？"
- "请用更简单的语言解释"
- "请对比方案A和方案B的优劣"

## 验证 Claude 的输出

Claude 不是万能的。以下场景需要特别注意验证：

### 需要验证的情况

1. **事实性信息**：具体日期、数字、引用来源
2. **专业建议**：法律、医疗、财务建议仅供参考
3. **代码和技术方案**：需要实际测试
4. **最新信息**：Claude 有知识截止日期

### 验证方法

- 对关键数据点，用搜索引擎交叉验证
- 对专业问题，咨询相关领域专家
- 对重要决策，不要仅依赖 AI 的建议

## Claude 的局限性

### 不擅长的事情

- 预测未来（股票、考试结果等）
- 提供最新的新闻和实时信息
- 进行复杂的数学计算（可能出错）
- 替代专业的法律/医疗建议

### 擅长的事情

- 文字处理和创作
- 信息整理和分析
- 头脑风暴和创意
- 解释复杂概念
- 代码编写和调试
